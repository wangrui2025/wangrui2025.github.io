内容结构怎么组织？

技术 + 设计双轨的内容组织方式

双轨意味着要同时覆盖两个维度：

  • 设计决策：字体选择、颜色系统、间距逻辑、视觉层次
  • 技术决策：为什么选 Astro / Tailwind v4、FOUC 处理、CDN 策略

两种组织方式

A

按决策类型分（当前方式）

Philosophy / Tokens / Pages / Components / Timeline / Notes

技术决策散落在各个 section 里，不容易看出因果关系。

B

按"决策-结果"分组（新方式）

每个大决策单独一个区块：

  • 🎯 字体系统：为什么选 Source Serif + Inter？（设计 Why）
    怎么实现的？（技术 How）
  • 🎯 颜色系统：为什么用 #c96442 做 accent？
    Tailwind v4 怎么配置的？
  • 🎨 暗色模式：为什么自建而不套用系统？
    FOUC 怎么处理的？
  • ⚡ 性能策略：为什么 CDN 固定 SHA？
    Critical CSS 内联怎么做？

每个决策讲清楚 Why + How，适合深入阅读。
