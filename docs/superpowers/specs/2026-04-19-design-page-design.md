# Design 页面设计规范

> 最后更新：2026-04-19

## 1. 定位与目的

**定位**：像设计师的创作手记，揭秘"我当时为什么这样想"

**目标读者**：面试官、同行、HR — 任何想了解设计背后思考的人

**与首页的关系**：
- 首页 (/)：干净的最终结果，给所有人看，不需要解释
- Design (/design/)：背后的思考，给想知道"为什么"的人看

---

## 2. 内容模块

| 模块 | 内容方向 |
|------|---------|
| **Hero** | 一句话介绍这个页面的目的 |
| **字体系统** | 为什么选 Source Serif + Inter？它们解决了什么问题？ |
| **颜色系统** | accent 色 #c96442 为什么选这个？参考了谁？ |
| **间距与层次** | 标题/正文的比例逻辑，字间距的设计意图 |
| **暗黑模式** | 为什么自建而不套系统？FOUC 怎么处理？ |
| **技术选型** | 为什么选 Astro v6 + Tailwind v4？ |
| **性能策略** | CDN 固定 SHA 为什么重要？Critical CSS 怎么做？ |
| **演进时间线** | 设计决策的历史记录（不是 commit 日志） |
| **Favicon 设计** | 宝可梦画廊的灵感，怎么从 800+ 选中 Smeargle |
| **简历/CV 设计** | 为什么用这个模板？结构化数据的选择 |
| **图标选择** | 怎么选图标？参考了哪些设计系统？ |

---

## 3. 写作风格

### 第一人称叙事
每个模块用"我"开头，解释设计决策的思考过程：
- "我选择 Source Serif 做标题，是因为..."
- "我最初用的是灰色标题，但测试发现对比度不够..."

### 三个维度
每个设计决策都要回答：
1. **为什么这样选？**（设计动机）
2. **我参考了什么？**（设计来源/灵感）
3. **它解决什么问题？**（设计目的）

### 参考样式
类似 Stripe / Vercel 的设计系统文档，但更个人化、更随意

---

## 4. 内容组织方式

**按"决策-结果"分组**，而非按技术/设计分类

每个决策区块：
```
🎯 [决策名称]：为什么选这个？（Why）
   怎么实现的？（How）
```

示例：
- 🎯 **字体系统**：为什么选 Source Serif + Inter？（设计 Why） + 怎么实现的？（技术 How）
- 🎯 **暗黑模式**：为什么自建而不套用系统？（设计 Why） + FOUC 怎么处理的？（技术 How）

---

## 5. 更新协议

- **更新频率**：每周集中更新一次
- **触发条件**：新的设计决策（组件/样式/页面/令牌）
- **不触发**：typo 修正、text-only 修改、minor CSS 调整

---

## 6. 页面结构

```
/design/
├── Hero（一句话目的说明）
├── 字体系统
├── 颜色系统
├── 间距与层次
├── 暗黑模式
├── 技术选型
├── 性能策略
├── 演进时间线
├── Favicon 设计
├── 简历/CV 设计
└── 图标选择
```

---

## 7. 技术实现

- **框架**：Astro v6 + Tailwind CSS v4
- **内容**：直接在 `design.astro` 页面中编写 HTML
- **样式**：复用餐桌的 design tokens（`@theme` in `global.css`）
- **发布**：随首页一同构建和部署
