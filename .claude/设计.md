1. 从排版学与文档设计角度 (Typography & Document Design)
旁注排版 (Marginalia / Margin Layout)：这是传统排版学中的叫法。将简短的元数据（如会议简称、年份）像“旁注”一样独立放置在左侧边缘区域，而将主要的、大段的内容留给右侧主视觉区。

悬挂式布局 (Hanging Layout) 或 整体缩进式对齐：左侧的标签作为一个悬挂的“锚点”，右侧的所有内容（无论是标题换行、作者列表还是链接）都在同一条垂直基准线上进行对齐，互不干扰。

2. 从简历设计的通用术语角度 (Resume / CV Design)
两栏式条目布局 (Two-column Entry Layout)：这是学术简历中最标准的称呼。左栏固定且极窄，仅用于提供“标签”（如会议级别、日期），右栏提供具有层级关系的详细描述（标题、作者、附属链接）。

时间线布局 (Timeline Layout)：因为学术发表、教育经历和工作经验通常都带有强烈的时间属性，这种左侧剥离出年份/会议的排版，在视觉引导上起到了一种垂直时间线的梳理作用。

3. 从前端开发与 UI 语义角度 (Web Development & UI)
描述列表布局 (Description List Layout)：在 HTML 结构语义中，这种设计非常类似于 <dl> (Description List) 的表现形式。左侧的 [CVPR 2026] 相当于 <dt> (Term/术语)，右侧的论文详情则是 <dd> (Details/描述)。

键值对布局 (Key-Value Layout)：左侧是 Key（发表在何处），右侧是 Value（发表了什么内容）。

## CV 页面设计决策 (2026-04-03)

### 作者列表空格问题
- **问题**：Astro 模板中 author 元素之间有换行，HTML 渲染时会产生多余空格
- **解决**：将 `map` 回调压缩为单行，确保 author span 与 separator span 直接相连
- **代码**：`{pub.authors.map((author, i) => (<><span>...</span>{i < len - 1 && <span>，</span>}</>))}`

### 姓名加粗策略
- **顶部姓名** (`cv-name`)：保留加粗 — 表示页面主人
- **论文作者** (`author-self`)：不加粗 — 与普通作者一致，仅用颜色区分

### 联系方式合并
- **之前**：分两行（电话丨地点）和（生日丨性别）
- **之后**：单行 + 新增 email 和 homepage
- **格式**：`15938550910 丨 深圳 丨 男（he/him）` / `wr_szu@163.com 丨 homepage`

### 荣誉奖项分级
- **主要荣誉**：黑色 (#1a1a1a)
- **次级荣誉**：灰色 (#999)，仅样式区分，不显示"次级"标签
- **数据字段**：`honors[]` 增加 `secondary?: boolean` 标志
